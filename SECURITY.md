# Security Policy

This repository contains public-facing doctrine, templates, and examples. It should not contain secrets, credentials, private client data, protected personal information, regulated advice, or sensitive operational details.

## Reporting sensitive issues

Use a private channel for anything sensitive.

Preferred path:

1. Use GitHub Security Advisories for this repository if enabled. If advisories are not enabled, do not post sensitive details publicly; open a minimal public issue requesting a private maintainer contact path.

Do not open a public issue for vulnerabilities, secrets, credentials, confidential business information, or private personal information.

## Public issue boundary

Public issues may be used for:

- Documentation improvements
- Template improvements
- Example improvements
- Public-safe business-system ideas
- Typo and formatting corrections

Public issues must not include:

- API keys, tokens, passwords, or credentials
- Internal company data
- Client names or confidential information
- Personal financial, health, legal, or identity information
- Proprietary strategy or sensitive operating details
