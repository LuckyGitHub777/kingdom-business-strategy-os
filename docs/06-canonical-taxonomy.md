# Canonical Taxonomy

This taxonomy aligns the doctrine, templates, examples, issue forms, and operating cadence. Use it when filing issues, reviewing decisions, writing examples, or extending the OS.

## The ten domains

| Domain | Definition | Typical evidence | Primary templates |
|---|---|---|---|
| Service / Customer | Who is served and what value they receive. | Customer interviews, payments, usage, renewals, referrals | [Customer Segment Brief](../templates/02_customer_segment_brief_template.md), [Business Model Review](../templates/01_business_model_review_template.md) |
| Stewardship / Resources | How money, time, people, tools, data, and opportunity are protected. | Budget, runway, resource plan, ownership map | [Business Model Review](../templates/01_business_model_review_template.md), [Decision Quality Checklist](../templates/04_decision_quality_checklist_template.md) |
| Cash | Whether obligations can be paid in real time. | Cash balance, burn, collections, payable schedule | [Forecast Assumptions Register](../templates/03_forecast_assumptions_register_template.md) |
| Margin / Economics | Whether value returned exceeds full delivery cost. | Gross margin, contribution margin, net margin, cost to serve | [Business Model Review](../templates/01_business_model_review_template.md) |
| Timing | Whether the action lands inside the real window. | Buying cycle, launch date, deadline, seasonality | [Decision Quality Checklist](../templates/04_decision_quality_checklist_template.md) |
| Forecasting | Whether expected outcomes are supported by evidence. | Base/downside/upside cases, assumptions, confidence | [Forecast Assumptions Register](../templates/03_forecast_assumptions_register_template.md) |
| Inventory / Capacity | Whether capacity is matched to credible demand. | Inventory, idle labor, unused tools, pipeline, utilization | [Forecast Assumptions Register](../templates/03_forecast_assumptions_register_template.md), [Operational Excellence Scorecard](../templates/05_operational_excellence_scorecard_template.md) |
| Operations / Quality | Whether promises can be delivered reliably. | Cycle time, rework, quality checks, SOPs, handoffs | [Operational Excellence Scorecard](../templates/05_operational_excellence_scorecard_template.md) |
| Trust / Governance | Whether the organization earns confidence and controls risk. | Policies, records, reviews, reputation, audit trail | [Decision Quality Checklist](../templates/04_decision_quality_checklist_template.md), [After-Action Review](../templates/06_after_action_review_template.md) |
| Learning Loop | Whether results become system updates. | After-action reviews, lessons, changed SOPs, training updates | [After-Action Review](../templates/06_after_action_review_template.md) |

## Mission and stewardship overlay

Some templates include an optional mission and stewardship alignment prompt. This is not an eleventh domain. It is a cross-check across the ten domains:

- Does the action serve a real customer or stakeholder?
- Does it protect resources, trust, and long-term value?
- Does it avoid extractive, confusing, or overpromised behavior?
- Does it create useful evidence for the next learning loop?

Use this overlay when the decision has brand, ethics, trust, public-use, or long-term legacy implications.
