# Contributing

Contributions should strengthen the operating doctrine, templates, examples, operating cadence, or implementation clarity.

## Contribution standards

- Keep claims practical, grounded, and evidence-aware.
- Avoid hype, manipulation, or guaranteed-outcome language.
- Prefer reusable templates, checklists, examples, and decision tools.
- Make lessons actionable for business owners, executives, and teams.
- Keep public-facing language professional, concise, and safe.
- Do not publish secrets, private business data, protected information, or confidential names.

## Suggested contributions

- New examples for the existing templates
- Improvements to the [Operating Cadence](docs/05-operating-cadence.md)
- Improvements to the [Canonical Taxonomy](docs/06-canonical-taxonomy.md)
- Forecasting examples
- Customer segment examples
- Operational excellence scorecards
- AI automation governance patterns
- Cyber/quantum/frontier-readiness business frameworks

## Contribution path

1. Read the [Quickstart](QUICKSTART.md).
2. Check the [Roadmap](ROADMAP.md) before adding a new artifact class.
3. Use the [Business System Improvement issue form](.github/ISSUE_TEMPLATE/business-system-improvement.yml) for proposed improvements.
4. Use the [Lesson Capture issue form](.github/ISSUE_TEMPLATE/lesson-capture.yml) to turn an outcome into doctrine, template, example, or training.
5. Submit a pull request using the [Pull Request Template](.github/pull_request_template.md).
